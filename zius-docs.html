Create Contact

🔒 Permission: create_contact
POST
/api/create/contact

Parameters

secret required string
Your API secret key from Tools → API Keys
phone required string
E.164 format (+201001234567) or Local format (01001234567)
name required string
Contact name
groups required string
Comma-separated list of group IDs
cURL 
curl -X POST "https://zius.uk/api/create/contact" \
     -H "Content-Type: multipart/form-data" \
     -F "secret=YOUR_API_SECRET" \
     -F "phone=+201001234567" \
     -F "name=John Doe" \
     -F "groups=1,2,3"
200 OK
JSON Response 
{
  "status": 200,
  "message": "Contact has been created!",
  "data": false
}

Send Single WhatsApp Message

🔒 Permission: wa_send
POST
/api/send/whatsapp

Required Parameters

secret required string
Your API secret key
account required string
WhatsApp account unique ID
recipient required string
Phone number (E.164 or local format) or group address
message required string
Message text or caption (supports Spintax)

Optional Parameters

type string
Message type: "text", "media", or "document" (default: "text")
priority number
1 for immediate send, 2 for normal (default: 2)
media_file binary
Media file for "media" type (jpg, png, gif, mp4, mp3, ogg)
media_url string
Direct URL to media file
media_type string
"image", "audio", or "video" (required with media_url)
cURL - Text Message 
curl -X POST "https://zius.uk/api/send/whatsapp" \
     -H "Content-Type: multipart/form-data" \
     -F "secret=YOUR_API_SECRET" \
     -F "account=WHATSAPP_ACCOUNT_ID" \
     -F "recipient=+201001234567" \
     -F "type=text" \
     -F "message=Hello! This is a test message."
200 OK
JSON Response 
{
  "status": 200,
  "message": "WhatsApp chat has been sent!",
  "data": {
    "messageId": 123
  }
}

Send USSD Request

🔒 Permission: ussd
POST
/api/send/ussd

Send USSD codes to check balance, activate services, and perform network operations.

Parameters

secret required string
Your API secret key
code required string
MMI request code (e.g., *123#)
sim required number
SIM slot number (1 or 2)
device required string
Device unique ID from /get/devices
cURL 
curl -X POST "https://zius.uk/api/send/ussd" \
     -H "Content-Type: multipart/form-data" \
     -F "secret=YOUR_API_SECRET" \
     -F "code=*123#" \
     -F "sim=1" \
     -F "device=00000000-0000-0000-d57d-f30cb6a89289"

🚀 Quick Start Guide

Step 1: Get Your API Credentials

  1. Sign up or login at zius.uk/dashboard
  2. Navigate to Tools → API Keys
  3. Generate or copy your API secret key
  4. Keep your API secret secure - never expose it in client-side code

Step 2: Connect Your Gateway

Choose one or both:

  • WhatsApp: Visit zius.uk/dashboard/hosts/whatsapp
  • Android SMS: Visit zius.uk/dashboard/hosts/android

Step 3: Make Your First API Call

Example: Send a WhatsApp message

JavaScript (Node.js) 
const FormData = require('form-data');
const axios = require('axios');

const formData = new FormData();
formData.append('secret', 'YOUR_API_SECRET');
formData.append('account', 'YOUR_WHATSAPP_ACCOUNT_ID');
formData.append('recipient', '+201001234567');
formData.append('type', 'text');
formData.append('message', 'Hello from Zius API!');

axios.post('https://zius.uk/api/send/whatsapp', formData, {
  headers: formData.getHeaders()
})
.then(response => {
  console.log('Success:', response.data);
})
.catch(error => {
  console.error('Error:', error.response.data);
});
Python 
import requests

url = 'https://zius.uk/api/send/whatsapp'
data = {
    'secret': 'YOUR_API_SECRET',
    'account': 'YOUR_WHATSAPP_ACCOUNT_ID',
    'recipient': '+201001234567',
    'type': 'text',
    'message': 'Hello from Zius API!'
}

response = requests.post(url, data=data)
print(response.json())

Step 4: Best Practices

  • ✅ Always use HTTPS for API requests
  • ✅ Implement proper error handling
  • ✅ Use E.164 phone number format (+countrycode + number)
  • ✅ Test with small groups before bulk campaigns
  • ✅ Monitor delivery status through the dashboard
  • ✅ Use Spintax to randomize messages and avoid spam filters
  • ✅ Set up webhooks for automated responses
  • ⚠️ Never expose API keys in client-side code
  • ⚠️ Respect rate limits to avoid throttling
